# contributing

contributions are welcome. this is a learning repository, so improvements to code quality, documentation, and examples are appreciated.

## how to contribute

1. fork the repository
2. create a feature branch (`git checkout -b feature/improvement`)
3. make your changes
4. ensure code follows go conventions (`go fmt ./...`)
5. commit with clear messages
6. push and open a pull request

## code standards

- follow [effective go](https://go.dev/doc/effective_go) guidelines
- add comments for exported functions
- include README updates for new concepts
- keep examples simple and focused

## areas for contribution

- additional examples for existing concepts
- new modules (interfaces, concurrency, testing)
- improved documentation
- bug fixes
- performance optimizations

## questions?

open an issue for discussion before major changes.
